# 产业链卡点研究库

用 **Serenity 方法**(源头证据、卡点优先)产出的产业链/单公司研究的归档与索引。
所有分析以 Markdown 存放,通过一个**纯静态、离线、零依赖**的生成器汇总成可浏览的网页门户。

> 免责声明:本研究库为公开资料的研究支持与排序,**不构成任何买卖建议**;决策与风险由使用者自行把握。

---

## 目录结构

```
industryresearch/
├── README.md          # 本说明
├── library.json       # 目录元数据(新增分析时改这里)
├── build.py           # 静态站点生成器(只读 md,不修改正文)
├── build.sh           # 一键重建并打开(./build.sh)
├── index.html         # ← 门户首页 · 按行业浏览(双击用浏览器打开)
├── tickers.html       # 按标的反查视图(一只票出现在哪些分析)
├── md/                # 分析正文(所有 .md 放这里)
│   ├── 氮化镓(GaN)产业链深挖.md
│   ├── CRO产业链深挖.md
│   ├── 生命科学上游试剂产业链深挖.md
│   ├── 色谱分离纯化产业链深挖.md
│   ├── 工业气体产业链深挖.md
│   ├── 广钢vs华特(地缘氦气)深挖.md
│   ├── 键凯科技深挖.md
│   ├── 凯莱英深挖.md
│   └── 纳微科技深挖.md
└── pages/             # 生成的逐篇详情页(自动产出,勿手改)
```

职责分离:**`md/` = 内容,`library.json` = 目录,`build.py` = 生成器,`index.html`/`pages/` = 产出。**

## 如何查看

直接用浏览器打开 `index.html`(双击即可,无需服务器/联网):

```bash
open industryresearch/index.html        # macOS
# 或把 index.html 拖进浏览器
```

门户有两个视图,右上角可切换:

**① 按行业(`index.html`)**
- **按行业归纳**(半导体、医药生物…),每个行业内再分「产业链」与「单公司深挖」;
- **搜索框** + 行业筛选(按标题/标的/关键词实时过滤);
- **卡片**显示一句话判断、子板块、标的、日期,点「查看完整分析 →」进详情页;
- **详情页**渲染 md 全文(标题/表格/列表/引用/链接),带返回、面包屑、相关分析互链;
- **🎯 真卡点信号**(见下)金色高亮。

**② 按标的反查(`tickers.html`)**
- 列出每只标的**出现在哪些分析里**,点进各篇;
- 按 **🎯 星数**(被几条产业链列为头号卡点)→ 出现篇数 → 名称排序,**真卡点金色置顶**;
- 自动合并 A/H 同名(如凯莱英 002821 与 06821.HK);带搜索框。

## 新增一篇分析(3 步)

1. 把新的 `.md` 放进 **`md/`** 子目录;
2. 在 `library.json` 的 `entries` 里加一条(`match` 填文件名里的**唯一关键词**);
3. 重新运行生成器(任选其一):

```bash
./industryresearch/build.sh         # 一键重建 + 自动打开 index.html
./industryresearch/build.sh -n      # 只重建,不打开
python3 industryresearch/build.py   # 等价(不自动打开)
```

生成器只读 `md/`,不会改动你的正文,并会刷新 `index.html`、`tickers.html` 与 `pages/`。

### 行业归类约定(产业链 / 单公司 / 对比 通用)

- **把涉及的行业挂全。** 一篇分析往往横跨多条产业链——选一个**主行业**填 `industry`(决定卡片归在哪个分区),其余相关行业全部放进 `also_in`(如南大光电 `industry:化工新材料` + `also_in:["半导体"]`;广钢vs华特同理)。点任一相关行业的筛选钮都能显示它,卡片上标「↳ 也属:X」。
- **出现新行业 → 直接加进主页。** 若涉及的行业还没在 `industries_order` 里(如未来的「高端制造」「电力设备」),就在 `library.json` 的 `industries_order` 数组里**加上该行业名**——它会自动获得筛选钮、独立分区和一个默认灰色圆点。想要专属圆点色,可选地在 `build.py` 的 CSS 里加一行 `.dot.行业名{background:#色值}`(不加也能正常显示)。
- 这套约定对**产业链、单公司深挖、对比深挖**一视同仁。

## `library.json` 字段

顶层:

| 字段 | 说明 |
|---|---|
| `site_title` / `site_subtitle` | 门户标题与副标题 |
| `industries_order` | 行业展示顺序(如 `["半导体","医药生物"]`) |
| `type_order` | 类型展示顺序(如 `["产业链","单公司深挖","对比深挖"]`) |
| `entries` | 每篇分析一条(见下) |

每条 `entry`:

| 字段 | 说明 |
|---|---|
| `match` | 文件名里的唯一关键词,生成器据此把 entry 匹配到 `md/` 下的文件(对大小写/全半角括号不敏感,稳妥) |
| `title` | 卡片与页面标题 |
| `industry` | **主**行业大类(决定卡片归在哪个分区,建议与 `industries_order` 对齐) |
| `also_in` | **(可选)** 跨行业标签数组(如 `["半导体"]`)。卡片仍住在主行业分区,但点这些行业的筛选钮也会显示它,并标「↳ 也属:X」。用于一篇分析横跨多条产业链的情形(如南大光电=化工新材料,也属半导体) |
| `subsector` | 子板块(如「生命科学上游 · 色谱填料」) |
| `type` | `产业链` / `单公司深挖` / `对比深挖` |
| `tickers` | 标的数组,格式 `"公司名 代码"`(如 `"纳微科技 688690"`、`"英诺赛科 02577.HK"`) |
| `key_pick` | **(仅产业链)** 该条产业链识别出的**头号卡点标的公司名**(如 `"广钢气体"`)。驱动 🎯 真卡点信号(见下),非产业链可省略 |
| `date` | `YYYY-MM-DD` |
| `summary` | 一句话判断(卡片摘要) |
| `related` | 关联分析的**文件名(去掉 .md)**数组,渲染为详情页底部互链 |

生成器会对不匹配项给出 `[warn]`(md 没有对应 entry,或 entry 没有对应 md),便于排查。

## 🎯 「真卡点信号」是怎么算的

本库的核心方法论标记,**由 `key_pick` 自动驱动、统一金标**:

> **某标的被几条产业链列为头号卡点(`key_pick`),就给它几颗 🎯。**
>
> - **1 颗 🎯 = 层内卡点**:某条产业链识别出的头号卡点标的;
> - **≥2 颗 🎯 = 多源指向**:多条**相互独立**的产业链共同把它列为头号卡点——Serenity 方法里最强的信号(值得优先深挖,也最该警惕拥挤)。

- 按**公司名**归并(自动合并 A/H 同名,如 `凯莱英 002821` 与 `凯莱英 06821.HK`);
- 命中后多处高亮:首页顶部金色横幅(按星数置顶排序)、产业链卡片的「🎯 本链头号卡点」标、头号卡点标的的单公司卡片金色光环+「N×🎯」标签、所有出现处的标的 chip 变金色、详情页顶部信号框、按标的反查里金色置顶;
- 想新增一个头号卡点:在该产业链 entry 里写 `"key_pick": "公司名"` 即可;同一标的被多条链设为 `key_pick`,星数自动累加、自动从层内卡点升级为多源指向。

当前命中:**纳微科技 🎯🎯**(生命科学上游试剂 + 色谱分离纯化,多源指向)、**广钢气体 / 键凯科技 / 英诺赛科 各 🎯**(各自所在产业链的层内卡点)。库变大后,新的收敛标的会自动升星。

> 注:`key_pick` 是"某条链的头号卡点"这一**人工研究判断**(写在 library.json),不是机械统计;这样金标既稀有又有信息量。

## 环境要求

- 仅需 **Python 3**(标准库即可,**无第三方依赖**);
- 内置轻量 Markdown→HTML 转换器(支持标题、GFM 表格、有序/无序列表含嵌套、引用、分隔线、加粗、行内代码、链接);
- 产出为纯静态 HTML,可整目录拷走或丢进任意静态托管。
